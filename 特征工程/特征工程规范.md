# 特征工程规范 — 网络正则化 VARX 预测

## 1. 目标

从阶段二产出的 GMVP 权重序列和资产网络中构建可预测未来权重的特征矩阵，使下游 VARX 模型实现：

| 指标 | 目标 | 说明 |
|------|------|------|
| MSE | ≤ 1× baseline（VAR无网络正则化） | 网络特征必须提供增量信息 |
| 网络特征排名 | 前 10 特征中至少 2 个出自网络拓扑 | 度/PageRank/聚类系数需有区分度 |
| 组合方差（OOS） | 相对 GLasso 静态权重不退化 | 预测权重比静态权重更优或持平 |
| 换手率（预测） | ≤ 1.5 倍静态 GMVP 换手率 | 避免过度交易 |

## 2. 待调参数

共 **9 个参数待定**，分为两个层级：

### 2.1 特征工程级（影响 X 的结构）

| 参数 | 默认值 | 扫描范围 | 说明 |
|------|:--:|:--:|------|
| `P_LAGS` | 3 | 1, 2, 3, 5 | 滞后阶数。超过 5 会导致 K×P > 训练样本 |
| `NETWORK_THRESHOLD` | 0.7 | 0.5, 0.7, 0.8 | Ā 截断阈值。0.3→100%密度(无效), 0.7→25%密度(度~98) |
| **`Ā_ROLLING_WINDOW`** | **20** | **10, 20, 30, 60** | Ā 滚动平均天数。越长=网络变化越平滑 |

### 2.2 模型级（影响 VARX 拟合）

| 参数 | 默认值 | 扫描范围 | 说明 |
|------|:--:|:--:|------|
| `LAMBDA_LASSO` | 1e-4 | 1e-5 ~ 1e-3 | ℓ1 系数稀疏惩罚 |
| `LAMBDA_TURNOVER` | 1e-2 | 0, 1e-3, 5e-3, 1e-2 | 换手率平滑 ℓ2。0=不惩罚 |
| `LAMBDA_NETWORK` | 1e-3 | 0, 1e-4, 1e-3, 1e-2 | 网络正则化强度。0=无网络引导 |
| `L_TRAIN_VARX` | 500 | 250, 500, 750 | 训练窗长（天） |
| `L_VAL_VARX` | 60 | 40, 60, 90 | 验证窗长（天） |
| `L_TEST_VARX` | 200 | 150, 200, 300 | 测试窗长（天） |




### 2.3 调优规程

1. 先用默认值跑通全流程
2. 固定窗网格搜索：`P_LAGS` × `LAMBDA_LASSO` × `NETWORK_THRESHOLD` × `Ā_ROLLING_WINDOW`
3. 最佳参数组合确定后，再扫描 `LAMBDA_TURNOVER` 和 `LAMBDA_NETWORK`
4. 最后微调时间窗 `L_TRAIN_VARX` / `L_VAL_VARX` / `L_TEST_VARX`

## 3. 需求

### 3.1 特征构建

- 不做手工剔除（保留原始维度供网络正则化约束）

### 4.3 数据隐私

全部使用公开市场数据（S&P 500 收益率 + CBOE VIX），无隐私限制。

## 5. 验收标准

### 5.1 特征有效性

| 指标 | 要求 | 方法 |
|------|:--:|------|
| 方差阈值 | 无全零特征 | `np.std(X, axis=0) > 1e-6` |
| 网络特征重要性 | LASSO 系数非零 ≥ 2 个 | `lasso.coef_[网络特征位置] ≠ 0` |
| VIX 特征重要性 | LASSO 系数非零 | `lasso.coef_[VIX位置] ≠ 0` |
| 滞后权重稀疏性 | ≥ 50% 系数为 0 | P_LAGS × K 维度中至少一半被 LASSO 压缩 |

### 5.2 分布稳定性

| 指标 | 要求 | 方法 |
|:--:|:--:|------|
| PSI (群体稳定性) | ≤ 0.5 通过，报告 p50/p95 | 分位数分箱（基于训练集decile边缘），比例制PSI |
| 均值偏移 | 每个特征 `max(μ_test/μ_train, μ_train/μ_test) < 2` | 训练集 vs 测试集均值比 |
| 缺失率 | 0% | 全部特征无 NaN |

> **PSI 阈值说明**：PSI ≤ 0.1 是信用评分模型（6-12月窗口）的监控标准，**不适用于本项目**跨10年+COVID崩盘的金融时序数据。理由：
>
> | PSI值 | 在本项目中的含义 | 处理 |
> |:--:|------|------|
> | ≤ 0.1 | 特征分布几乎不变（如部分GMVP权重） | 安全，但可能无预测增量信息 |
> | 0.1 ~ 0.5 | 适中漂移，捕捉了市场结构变化 | **正常，这是预测信号的来源** |
> | 0.5 ~ 2.0 | 明显漂移（如网络聚类系数） | 可接受，指数崩塌/重连本身就是信息 |
> | > 2.0 | 剧烈漂移（如VIX从15飙到80） | **保留**，这是最强外生信号，删掉反而伤模型 |
>
>
> **当前基线**（P_LAGS=3, NETWORK_THRESHOLD=0.3, Ā_WINDOW=20）：
> - PSI p50 = 0.35（一半特征 ≤ 0.35）
> - PSI p95 = 1.83
> - PSI ≤ 0.5 通过率 ≈ 60%

### 5.2.1 PSI 与模型预测能力的关系

高 PSI ≠ 坏特征。在金融预测场景中，分布的**时变性本身就是预测信号**：
- VIX PSI ≈ 12：训练期均值 15（QE时代），测试期飙至 80（COVID）。保留它是必须的——VIX 是 VARX 中最强的外生变量
- 网络度中心性 PSI 中等：崩盘期网络密度剧烈变化，反映系统性风险传导路径的改变
- 若强行将 PSI 压到 ≤0.1（如标准化、加窗平滑），反而会**抹掉预测信号**，导致 MSE 升高

**验收逻辑**：PSI 报告的是"分布漂移了没有"，不是"漂移好不好"。真正的好/坏由下游 VARX 的 LASSO 系数和 OOS MSE 判定。PSI 超标不触发退化回退。

### 5.3 无数据泄露验证

1. 时间顺序检查：`max_train_date < min_valid_date < min_test_date` ✅
2. 外生变量滞后检查：VIX 在 t 时刻预测 `w̃_t`，t 时刻 VIX 收盘价在 A 股交割前已知 ✅（美东时间下午 4:00 = 北京时间次日凌晨 4:00）
3. 网络特征检查：`Ā_{t-1}` 只用 t-1 及之前数据 ✅

### 5.4 退化检查

若任一验收指标不达标，回退方案：
- 删网络特征 → 退化为 sparse VARX（仅滞后权重 + VIX）
- 删 VIX → 退化为 sparse VAR
- 删所有外生变量 → 退化为基础 VAR

# VARX 预测规范（对齐论文 Table 2）

## 1. 目标

对齐论文 §5.2 Table 2 的 7 个模型 + 4 个评估指标：

**评估指标**（论文 Table 2）：

| 指标 | 含义 | 计算方式 |
|------|------|------|
| **MSE** | 预测权重与 realized GMVP 权重的均方误差 | `mean((w̃_t^pred - w̃_t)^2 / K)` |
| **MAE** | 平均绝对误差 | `mean(|w̃_t^pred - w̃_t| / K)` |
| **DM statistic** | Diebold-Mariano 检验（vs baseline VAR） | 正值表示优于 baseline |

| 2 | Sparse VAR | 滞后权重 | ✅ | | | |
| 3 | Sparse VARX | 滞后权重 + 外生（含网络拓扑） | ✅ | | | |
| 4 | **Network VARX** | 同 M3（同 x_t，加 λ₁≠λ₂ 网络惩罚） | ✅ | ✅ | | |
| 5 | Network VARX + Smooth | 同上 | ✅ | ✅ | ✅ | |
| 6 | Network VARX + DFL | 同上 | ✅ | ✅ | | ✅ |
| 7 | LSTM | 滞后权重（深度学习 baseline） | — | — | — | — |

**目标约束**：

| 约束 | 要求 |
|------|------|
| Model 4(Network VARX) 的 MSE 需显著低于 Model 3(Sparse VARX) | 网络特征必须提供增量信息 |

## 2. 模型公式

### 2.1 论文 §3.2 主模型

```
w̃_t = B₁·w̃_{t-1} + ... + Bₚ·w̃_{t-p} + C·x_t + ε_t
```

| 符号 | 含义 | 维度 |
|------|------|:--:|
| w̃_t | t 日 GMVP 权重（目标） | K=392 |
| B₁…Bₚ | 滞后权重系数 | K×K（P_LAGS 个） |
| x_t | 外生变量（VIX + BA spread + 网络拓扑） | n_exog ≈ 1,178 |
| C | 外生系数 | K×n_exog |

### 2.2 简化：逐资产 Lasso

K=392 → 全矩阵 B（153,664 系数×P_LAGS）不可行。改为：

```
w̃_{t,i} = β_iᵀ·滞后权重 + γ_iᵀ·x_t + ε_{t,i}
```

每支股票 i 独立 fit Lasso，系数总数降至 392×(P_LAGS×K+n_exog) ≈ 每支 2,354 个。

### 2.3 求解方法与损失函数

**两步法**：构建网络掩码 → 逐资产加权 Lasso。

**目标函数（对齐论文公式 13）**，对全系统系数矩阵 `Θ = [B₁…B_p, C₀…C_q]`：

$$\begin{aligned}
\hat{\Theta} = \arg\min_{\Theta} \Bigg(
  &\frac{1}{T_0} \sum_{t \in \mathcal{T}_{\text{train}}} \|\tilde{\mathbf{w}}_{t+1} - \hat{\mathbf{w}}_{t+1}^{\text{raw}}(\Theta)\|_2^2 \\\\
  &+ \lambda_1 \sum_{i=1}^{K} \sum_{\ell=1}^{p} \sum_{j \in \mathcal{N}_i} |B_{\ell,ij}|
   + \lambda_2 \sum_{i=1}^{K} \sum_{\ell=1}^{p} \sum_{j \notin \mathcal{N}_i, j \neq i} |B_{\ell,ij}|
   + \lambda_3 \sum_{\ell=0}^{q} \|C_\ell\|_1 \\\\
  &+ \lambda_s \sum_{t \in \mathcal{T}_{\text{train}}} \|\hat{\mathbf{w}}_{t+1}(\Theta) - \hat{\mathbf{w}}_t(\Theta)\|_2^2
\Bigg)
\end{aligned}$$

| 符号 | 含义 | 参数值 | 工作 |
|:--:|------|:--:|------|
| T₀ | 训练样本数 | 1690 | |
| λ₁ | 网络内连接资产的滞后系数 ℓ1 惩罚 | `LAMBDA_LASSO = 1e-4` | 轻，鼓励跨资产依赖 |
| λ₂ | 网络外资产的滞后系数 ℓ1 惩罚 | `LAMBDA_LASSO + LAMBDA_NETWORK = 1.1e-3` | 重（>λ₁），压制噪声连接 |
| λ₃ | 外生变量+网络拓扑系数 ℓ1 惩罚 | `LAMBDA_EXOG = 1e-4` | x_t 统一惩罚，独立于 λ₁ |
| λ_s | 换手率平滑 ℓ2 惩罚 | `LAMBDA_TURNOVER = 1e-2` | 仅 Model 5 |
| 𝒩_i | 资产 i 的网络邻居 | `{j : Ā[i,j] ≥ NETWORK_THRESHOLD}` | |

> **核心直觉**：λ₂ > λ₁，未连接资产受到 11 倍惩罚，其系数被强烈压向零。只有网络中真正相连的资产对之间的滞后效应才被 Lasso 保留。这就是"网络结构引导稀疏模式"。

#### 2.3.1 实现：逐资产加权 Lasso

全系统联合估计（K×K 个交互系数）不可行。改为对每个资产 i 独立求解，系数维度从 O(K²) 降至 O(K)：

$$\hat{\beta}_i = \arg\min_{\beta_i} \frac{1}{T_0}\sum_t (w_{t,i} - \mathbf{z}_t^\top\beta_i)^2 + \sum_{j=1}^{d} \alpha_{ij}|\beta_{ij}|$$

其中特征 j 的惩罚 α_{ij} 取决于 j 对应的资产是否在 𝒩_i 中：

| 特征类型 | α_{ij} | 对应符号 |
|---|---|---|
| 滞后权重，j ∈ 𝒩_i | `LAMBDA_LASSO` | λ₁ |
| 滞后权重，j ∉ 𝒩_i | `LAMBDA_LASSO + LAMBDA_NETWORK` | λ₂ |
| 外生变量（宏观） | `LAMBDA_EXOG` | λ₃ |
| 网络拓扑特征 | `LAMBDA_EXOG` | λ₃（属 x_t 的一部分） |

#### 2.3.2 网络自适应惩罚

sklearn 只支持统一 α。处理方式：基于构造的网络惩罚向量，对特征 Xⱼ 按 sⱼ = λ₁/αⱼ 缩放后再跑标准 Lasso，此时系数会放大，故惩罚会放大，从而将网络结构考虑进惩罚，拟合后系数乘以 sⱼ 还原——缩放即可将不同惩罚"编码"进数据，等价于加权 L1。代码见 `VARX预测.py` 的 `_fit_one_asset()`。

#### 2.3.3 换手率平滑 λ_s 的实现：数据增广法

Model 5 的 λ_s 项 `λ_s · Σ_t ‖ŵ_{t+1} - ŵ_t‖²` 通过**数据增广**嵌入训练：

```
追加行:  √(T₀·λ_s)·ΔX   目标: 0
其中 ΔX = X_{t+1} - X_t 逐日特征差分, T₀ = 训练样本数
```

sklearn Lasso 的 `(1/(2n))` 归一化展开后，平滑项与 MSE 的权重比 = `T₀·λ_s`，与论文公式(13)一致。`LAMBDA_TURNOVER` 控制 λ_s 的强度。代码见 `VARX预测.py` 的 `fit_model()`。

**Model 6（DFL）** 用决策聚焦损失替代 MSE — 见 `决策聚焦训练.py`：

$$\mathcal{L}_{\text{DFL}}(\Theta) = \frac{1}{T_0}\sum_t \left( \hat{\mathbf{w}}_{t+1}^\top \hat{\Sigma}_t^{\text{RC}} \hat{\mathbf{w}}_{t+1} + \eta \|\hat{\mathbf{w}}_{t+1} - \hat{\mathbf{w}}_t\|_1 \right)$$

其中 Σ̂ 是已实现协方差矩阵，η 是交易成本系数。

### 2.4 网络均值 Ā 的两种选择

| 方案 | 做法 | 适用场景 |
|------|------|------|
| **固定窗滚动**（当前） | `Ā_ROLLING_WINDOW=20` 天滑动平均 | 捕捉时变网络结构 |
| **全样本均值** | 全部 2425 天邻接矩阵等权平均 | 网络结构稳定时更简洁 |

当前采用滚动 20 天（论文公式 7），待定。

| `LAMBDA_TURNOVER` | 1e-2 | 0, 1e-3, 5e-3, 1e-2 | 换手率平滑 ℓ2（Model 5） |
| `LAMBDA_NETWORK` | 1e-3 | 0, 1e-4, 1e-3, 1e-2 | 网络正则化强度（Model 4-6） |
| `ETA`（DFL） | 1e-4 | 0, 1e-5, 1e-4, 1e-3 | 决策聚焦的换手率惩罚（Model 6） |
| 时间窗 | 500/60/200 | 250~750 / 40~90 / 150~300 | T/V/Test |

**调优规程**：从 Model 1 开始逐层叠加 → 每步固定上层超参数，网格搜索新增参数。

## 4. 验收标准

| 检查项 | 要求 | 方法 |
|:--:|:--:|------|
| **无数据泄露** | max(train) < min(val) < min(test) | 日期严格检查 |
| **DM 检验** | Model 4-6 vs Model 1 显著 | `dm_test()` p < 0.05 |
| **MCS 排名** | 自评方差最小的 3 个模型进入 MCS | Hansen(2005) 程序 |
| **净 Sharpe** | Model 6 ≥ Model 4 | 扣除 10 bps 双边交易成本 |
| **换手率** | Model 5 换手率 ≤ 1.5× Model 4 | 回测计算 |
| **系数稀疏性** | ≥ 50% 滞后系数 = 0 | Lasso 截断 |
| **PSI** | ≤ 0.5（参见 §5.2） | 分位数分箱，比例制 PSI |

> **PSI 阈值**：§5.2 已将 ≤0.1 修正为 ≤0.5，并新增 p50/p95 报告。金融跨10年+COVID数据中 0.1 不可行。PSI 超标不触发退化回退。

> **验证集**：当前固定切分中包含验证集（362天），但参数调优阶段未启用。将在网格搜索 `P_LAGS × LAMBDA_LASSO × NETWORK_THRESHOLD` 时接入。
